<objective>
Land the cryptographic and conformance foundation for the entire suite. This plan ships:

1. **`IdentityProvider` Protocol** with the locked method signatures every adapter must implement (IDENT-01).
2. **Brine reference adapter** — real PyNaCl Ed25519 with `python-keyring` for key material; never returns key bytes; refuses to start without keystore (IDENT-02, IDENT-05; Pitfall 9).
3. **Verifier with key-scheme dispatch** — refuses scheme/signature mismatch (IDENT-03).
4. **`Receipt` value type with private constructor** — frozen dataclass + module-private sentinel; only `IdentityProvider.verify` returning success can produce one. Direct construction fails at runtime AND under mypy --strict (D-01; IDENT-04; Pitfall 5).
5. **Initial conformance fixture set** — manifest schema (D-04) plus one valid `(task, task_result)` pair plus six invalid fixtures, one per Thermocline AT-C surface (AT-C1..AT-C6) (THERMO-06).
6. **Spec patch tracking** — `thermocline/CHANGELOG.md` records the cirdan→thermocline rename and any patches discovered during Phase 1 implementation (THERMO-01).

Purpose: This is where the privacy-receipt threat model becomes code. Every other layer of the suite — Photophore's audit log, the dispatch coordinator, the upgraded forges — sits on top of these primitives. The Receipt private-constructor mechanism makes "skipped verification" inexpressible. The brine adapter's refusal to fall back to file storage makes "convenient" key handling impossible. The conformance fixtures give every future implementation (`thermocline-ts`, `thermocline-rs`, third-party forges) a concrete contract to validate against.

Output: `thermocline.identity` module with `IdentityProvider`, `BrineProvider`, `Receipt`, `Verifier`; full unit + integration + property tests including the static type-check fixture; six AT-C invalid fixtures + valid pair + manifests under `thermocline/conformance/`; CHANGELOG entry plus any spec amendments.

```python
class Signature:
    """Opaque value returned by IdentityProvider.sign — wraps the raw signature bytes."""
    def __init__(self, *, scheme: KeyScheme, bytes_: bytes, signer_identity: str) -> None: ...
    @property
    def scheme(self) -> KeyScheme: ...
    @property
    def bytes_(self) -> bytes: ...
    @property
    def signer_identity(self) -> str: ...

class Receipt:
    """Verification witness. Constructible ONLY via IdentityProvider.verify() returning success.

    Direct construction (Receipt(envelope_id=..., ...)) raises TypeError at runtime AND
    fails mypy --strict because the only public __init__ takes a sentinel token that's
    not in the public namespace.
    """
    @property
    def envelope_id(self) -> str: ...
    @property
    def signature_hash(self) -> str: ...  # hex digest of canonical-JSON+signature
    @property
    def verified_at(self) -> datetime: ...
    @property
    def key_scheme(self) -> KeyScheme: ...

class IdentityProvider(Protocol):
    scheme: ClassVar[KeyScheme]
    def sign(self, *, envelope: dict, signer_identity: str) -> Signature: ...
    def verify(self, *, envelope: dict, signature: Signature) -> Receipt | None: ...
    def public_key(self, *, identity: str) -> bytes: ...
    def generate(self, *, identity: str) -> None: ...

class BrineProvider:
    """Reference IdentityProvider for the brine (Ed25519 + RFC 8785 + python-keyring) scheme.

    Refuses to start if the platform secure keystore is unavailable (IDENT-05).
    Never copies key material out of the keystore beyond the keystore-RPC's transient
    return (IDENT-02 + Pitfall 9).
    """
    scheme: ClassVar[KeyScheme] = KeyScheme.BRINE
    def __init__(self, *, keyring_service: str = "thermocline.brine") -> None: ...

class Verifier:
    """Multi-scheme verifier. Dispatches on envelope.key_scheme to the registered provider."""
    def __init__(self) -> None: ...
    def register(self, provider: IdentityProvider) -> None: ...
    def verify(self, *, envelope: dict, signature: Signature) -> Receipt | None: ...
```

3. **`_RECEIPT_TOKEN`** — module-private sentinel. Implementation:
   ```python
   class _ReceiptConstructorToken:
       """Module-private witness that Receipt construction came from a verified path.

       The class is defined in this module and never exported. Any caller that wants
       to construct a Receipt must pass an instance of this class — but the class is
   ```

4. **`Receipt`** — D-01 mechanism. Frozen `@dataclass(frozen=True, slots=True)` with `envelope_id: str`, `signature_hash: str`, `verified_at: datetime`, `key_scheme: KeyScheme`. Override `__init__` to require the sentinel:
   ```python
   from dataclasses import dataclass, field
   from datetime import datetime
   from typing import Final, ClassVar

   @dataclass(frozen=True, slots=True)
   class Receipt:
       envelope_id: str
       signature_hash: str
       verified_at: datetime
       key_scheme: KeyScheme

       def __init__(
           self,
           *,
           envelope_id: str,
           signature_hash: str,
           verified_at: datetime,
           key_scheme: KeyScheme,
           _token: "_ReceiptConstructorToken | None" = None,
       ) -> None:
           if _token is not _RECEIPT_TOKEN:
               raise TypeError(
                   "Receipt is constructible only by IdentityProvider.verify() returning success. "
                   "Direct construction is forbidden by design (D-01); see "
                   ".planning/phases/01-thermocline-py-foundations/01-CONTEXT.md."
               )
           # frozen=True forbids attribute assignment; use object.__setattr__.
           object.__setattr__(self, "envelope_id", envelope_id)
           object.__setattr__(self, "signature_hash", signature_hash)
           object.__setattr__(self, "verified_at", verified_at)
           object.__setattr__(self, "key_scheme", key_scheme)
   ```

   Static-type story: the explicit `__init__` signature has `_token: "_ReceiptConstructorToken | None" = None`, but `_ReceiptConstructorToken` is module-private. `mypy --strict` on a module that does `Receipt(envelope_id=..., signature_hash=..., verified_at=..., key_scheme=...)` (without `_token`) will pass (the default makes `_token` optional)... so the static check needs another layer. Two options:

   - **Option A** (chosen): rename the sentinel to `__token` (double-underscore reserved) and use `Final` typing, then add a `# type: ignore[call-arg]` requirement in misuse fixtures that must be present for misuse to compile — too brittle.

   - **Option B** (chosen): make `_token` parameter REQUIRED (no default). External callers cannot type-check it because `_ReceiptConstructorToken` is module-private. mypy --strict on `tests/fixtures/receipt_misuse.py` (which directly does `Receipt(envelope_id="x", ...)`) fails with `Missing positional argument "_token" in call to "Receipt"`.

   **Use Option B.** Drop the `= None` default. The internal call site (`IdentityProvider.verify` returning a Receipt) passes `_token=_RECEIPT_TOKEN` explicitly.

   ```python
   def __init__(
       self,
       *,
       envelope_id: str,
       signature_hash: str,
       verified_at: datetime,
       key_scheme: KeyScheme,
       _token: "_ReceiptConstructorToken",   # <-- required, no default
   ) -> None:
       if _token is not _RECEIPT_TOKEN:
           raise TypeError("...")
       ...
   ```

   This satisfies BOTH the runtime check (foreign sentinel instances raise TypeError) AND the static check (mypy refuses `Receipt(..., ...)` without `_token` because `_ReceiptConstructorToken` is module-private and an external caller has nothing valid to pass).

6. **`Verifier`** — multi-scheme dispatch (IDENT-03):
   ```python
   class Verifier:
       def __init__(self) -> None:
           self._providers: dict[KeyScheme, IdentityProvider] = {}
       def register(self, provider: IdentityProvider) -> None:
           self._providers[provider.scheme] = provider
       def verify(self, *, envelope: dict, signature: Signature) -> Receipt | None:
           # Envelope's declared scheme MUST match the signature's scheme.
           declared = envelope.get("key_scheme")
           if declared != signature.scheme.value:
               raise SchemeError(
                   f"declared key scheme {declared!r} does not match signature scheme {signature.scheme.value!r}",
                   code="UNSUPPORTED_KEY_SCHEME",
               )
           provider = self._providers.get(signature.scheme)
           if provider is None:
               raise SchemeError(
                   f"no IdentityProvider registered for scheme {signature.scheme.value!r}",
                   code="UNSUPPORTED_KEY_SCHEME",
               )
           return provider.verify(envelope=envelope, signature=signature)
   ```

```python
import hashlib
from datetime import datetime, timezone
import keyring
from keyring.errors import NoKeyringError
import nacl.signing
import nacl.exceptions

from .canonical import canonicalize
from .errors import IdentityError, KeystoreUnavailableError, SchemeError

_KEYSTORE_SERVICE_DEFAULT: Final[str] = "thermocline.brine"


class BrineProvider:
    """Ed25519 IdentityProvider backed by python-keyring (Keychain / libsecret / Credential Manager).

    Properties (forever):
      - Never returns private key material from any method (IDENT-02).
      - Refuses to start when the platform keystore is unavailable (IDENT-05) — does NOT
        fall back to file-based or env-var-based key storage. There is no fallback path.
      - Calls keyring per signature; no in-process key cache (Pitfall 9).
      - Signing input is canonical-JSON via thermocline.canonical.canonicalize (Pitfall 11).
    """

    scheme: ClassVar[KeyScheme] = KeyScheme.BRINE

    def __init__(self, *, keyring_service: str = _KEYSTORE_SERVICE_DEFAULT) -> None:
        # IDENT-05: probe the keystore at startup; refuse if unavailable.
        try:
            backend = keyring.get_keyring()
            # Some backends report present but cannot actually access — the
            # null/fail backend is exposed as `keyring.backends.fail.Keyring`.
            backend_name = type(backend).__name__
            if "fail" in backend_name.lower() or "null" in backend_name.lower():
                raise KeystoreUnavailableError(
                    f"refusing to start: keyring backend is {backend_name!r} (no working secure keystore). "
                    "Brine adapter NEVER falls back to file or env-var storage (IDENT-05).",
                    code="KEYSTORE_UNAVAILABLE",
                )
        except NoKeyringError as exc:
            raise KeystoreUnavailableError(
                f"refusing to start: no keyring backend available ({exc}). "
                "Brine adapter NEVER falls back to file or env-var storage (IDENT-05).",
                code="KEYSTORE_UNAVAILABLE",
            ) from exc
        self._keyring_service = keyring_service

    def generate(self, *, identity: str) -> None:
        """Generate a new Ed25519 keypair and store the private seed in the keystore.

        The keystore returns nothing to us — we hand the seed directly to keyring.
        """
        signing_key = nacl.signing.SigningKey.generate()
        # Store the 32-byte seed (NOT the full SigningKey object) as a hex string.
        # python-keyring expects str passwords; we'll hex-encode the seed.
        keyring.set_password(self._keyring_service, identity, signing_key.encode().hex())
        # IMMEDIATELY drop our reference. Best-effort — Python doesn't guarantee
        # zeroing, but we do not retain the SigningKey or the seed beyond this scope.
        del signing_key

    def public_key(self, *, identity: str) -> bytes:
        seed_hex = keyring.get_password(self._keyring_service, identity)
        if seed_hex is None:
            raise IdentityError(
                f"no brine key stored for identity {identity!r}",
                code="IDENTITY_NOT_FOUND",
            )
        signing_key = nacl.signing.SigningKey(bytes.fromhex(seed_hex))
        verify_key_bytes = bytes(signing_key.verify_key)
        del signing_key  # drop reference
        return verify_key_bytes

    def sign(self, *, envelope: dict, signer_identity: str) -> Signature:
        """Sign canonicalize(envelope). Returns Signature; never returns key material."""
        seed_hex = keyring.get_password(self._keyring_service, signer_identity)
        if seed_hex is None:
            raise IdentityError(
                f"no brine key stored for identity {signer_identity!r}",
                code="IDENTITY_NOT_FOUND",
            )
        canonical = canonicalize(envelope)
        signing_key = nacl.signing.SigningKey(bytes.fromhex(seed_hex))
        signed = signing_key.sign(canonical)
        sig_bytes = signed.signature  # 64 bytes for Ed25519
        del signing_key  # drop reference (Pitfall 9)
        return Signature(scheme=KeyScheme.BRINE, bytes_=sig_bytes, signer_identity=signer_identity)

    def verify(self, *, envelope: dict, signature: Signature) -> Receipt | None:
        """Verify a brine signature. Returns Receipt on success, None on tamper.

        Raises SchemeError if the signature's declared scheme is not BRINE.
        """
        if signature.scheme is not KeyScheme.BRINE:
            raise SchemeError(
                f"BrineProvider cannot verify signature with scheme {signature.scheme.value!r}",
                code="UNSUPPORTED_KEY_SCHEME",
            )
        verify_key_bytes = self.public_key(identity=signature.signer_identity)
        verify_key = nacl.signing.VerifyKey(verify_key_bytes)
        canonical = canonicalize(envelope)
        try:
            verify_key.verify(canonical, signature.bytes_)
        except nacl.exceptions.BadSignatureError:
            return None
        sig_hash = hashlib.blake2b(canonical + signature.bytes_, digest_size=32).hexdigest()
        envelope_id = str(envelope.get("envelope_id", ""))
        return Receipt(
            envelope_id=envelope_id,
            signature_hash=sig_hash,
            verified_at=datetime.now(timezone.utc),
            key_scheme=KeyScheme.BRINE,
            _token=_RECEIPT_TOKEN,
        )
```

|----------|-------------|
| `IdentityProvider.sign` ↔ caller | Caller must NEVER receive raw key material; the sign API only returns Signature. Pitfall 9 enforces this at the API contract level. |
| `Receipt` construction ↔ verification path | Only `IdentityProvider.verify` returning success can produce a Receipt. Pitfall 5 — "skipped verification" must be inexpressible (D-01). |
